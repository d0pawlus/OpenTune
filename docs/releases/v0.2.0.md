# OpenTune v0.2.0 — M6

OpenTune v0.2.0 is the first public M6 release candidate. It focuses on
TunerStudio project interoperability, safe evaluation without hardware, a
repeatable cross-platform release pipeline, and an approachable first run.

## Highlights

- Compatibility evidence for Speeduino, rusEFI, and MegaSquirt MS3 definitions,
  including real TunerStudio-produced MSQ files and reproducible diagnostics.
- User-controlled in-app updates with a dedicated Tauri signature, explicit
  install/restart confirmation, retryable failures, and offline-safe startup.
- English/Polish onboarding with simulator, offline editing, and real-hardware
  safety guidance; persisted language and contrast preferences.
- Automated accessibility smoke checks, keyboard/focus regression coverage,
  shared 44 px targets, visible focus indicators, and reduced-motion support.
- macOS arm64/x64, Windows x64, and Linux x64 packages, `latest.json`, updater
  signatures, and `SHA256SUMS` verified by the release workflow.
- Public quick-start, installation/update, compatibility, accessibility,
  architecture, protocol, INI, roadmap, and ADR documentation.

## Important limitations

- macOS packages are not Apple-notarized and Windows packages are not
  Authenticode-signed. The project owner will add those publisher signatures
  after obtaining an Apple Developer account and Windows signing certificate.
  The Tauri updater signature is separate and is already enforced.
- M6 does not establish physical-ECU burn safety. Keep a known-good tune backup,
  use the exact firmware INI, and retain the vendor recovery path.
- Honda OBD1, BMW MS4x, Linux arm64, and unsupported TunerStudio
  `UserDefined` presentation widgets are not claimed by this release.
- The disposable OpenTune/TunerStudio GUI re-save and manual
  keyboard/high-contrast/VoiceOver checks remain explicit release gates until
  their results are added to the M6 evidence pages.

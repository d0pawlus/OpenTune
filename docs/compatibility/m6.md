# M6 compatibility evidence

Verified on 2026-07-19 on macOS 26.5.1. This report is an
interoperability snapshot, not a promise that every TunerStudio extension or
every physical ECU is supported. Paths beginning with `$HOME` name local,
non-redistributed TunerStudio projects; their SHA-256 hashes make the checks
repeatable without committing third-party tune files.

## Firmware matrix

| Family | Tested definition | Parsed model | INI diagnostics | MSQ load |
| --- | --- | --- | --- | --- |
| Speeduino | `speeduino 202504-dev`, upstream `noisymime/speeduino@0832dc1d`, committed golden fixture | 15 pages, 817 constants, 21 tables, 34 curves, 242 output channels, 95 gauges | 126 recorded/allowlisted degradations: 125 unsupported `UserDefined` widgets and one gauge configuration; the golden allowlist test passes | No matching real MSQ was distributed with this upstream definition; file-format serialization is covered by the focused round-trip test below |
| rusEFI | `rusEFI v20250618@1180919717`, signature `rusEFI .2025.12.13.zgrywus-k8.1180919717`, local Mazda Miata project, symbol `AFR` | 3 pages, 3694 constants, 85 tables, 70 curves, 982 output channels, 314 gauges | 970 non-fatal degradations, predominantly unsupported `UserDefined` presentation widgets (965); the definition remains usable | 3616 applied, 668 unknown/unallocated entries skipped, 20 values clamped to declared bounds, **0 failed** |
| MegaSquirt MS3 | TunerStudio `MS3-Example_Project`, firmware source path identifies MS3 1.3.0, signature `MS3 Format 0524.01` | 19 pages, 2306 constants, 83 tables, 132 curves, 462 output channels, 270 gauges | 411 non-fatal degradations, predominantly unsupported `UserDefined` presentation widgets (403); the definition remains usable | 1985 applied, 0 skipped, 1 value clamped to its declared bound, **0 failed** |

The diagnostic counts are intentionally visible. OpenTune degrades unsupported
TunerStudio-only presentation widgets instead of pretending to render them.
Hard INI parse errors and failed MSQ assignments are the M6 family gate; none
occurred in these three checks.

## Reproduction commands

Run from the repository root. The symbol lists are the `ecuSettings` values in
each TunerStudio project's `projectCfg/project.properties`.

```bash
cargo run -q -p opentune-ini --example dump \
  --manifest-path src-tauri/Cargo.toml -- \
  src-tauri/crates/ini/tests/fixtures/speeduino-real-0832dc1d.ini

cargo test -p opentune-ini --test real_ini \
  --manifest-path src-tauri/Cargo.toml

cargo run -q -p opentune-ini --example dump \
  --manifest-path src-tauri/Cargo.toml -- \
  "$HOME/TunerStudioProjects/MazdaMiata_DominikP_RusEfi_ZgrywusK8/projectCfg/mainController.ini" \
  AFR

cargo run -q -p opentune-project --example msq_dump \
  --manifest-path src-tauri/Cargo.toml -- \
  "$HOME/TunerStudioProjects/MazdaMiata_DominikP_RusEfi_ZgrywusK8/projectCfg/mainController.ini" \
  "$HOME/TunerStudioProjects/MazdaMiata_DominikP_RusEfi_ZgrywusK8/CurrentTune.msq" \
  AFR

MS3_SYMBOLS='CAN_COMMANDS_OFF,CYL_12_16_SUPPORT_OFF,FAHRENHEIT,INTERNAL_LOG_FIELDS_OFF,OUTMSG_EDITING_OFF,PORT_STATUS_OFF,PW_4X_OFF'
cargo run -q -p opentune-ini --example dump \
  --manifest-path src-tauri/Cargo.toml -- \
  "$HOME/TunerStudioProjects/MS3-Example_Project/projectCfg/mainController.ini" \
  "$MS3_SYMBOLS"

cargo run -q -p opentune-project --example msq_dump \
  --manifest-path src-tauri/Cargo.toml -- \
  "$HOME/TunerStudioProjects/MS3-Example_Project/projectCfg/mainController.ini" \
  "$HOME/TunerStudioProjects/MS3-Example_Project/CurrentTune.msq" \
  "$MS3_SYMBOLS"

cargo test -p opentune-project --test msq \
  tunerstudio_metadata_and_quoting_are_serialized \
  --manifest-path src-tauri/Cargo.toml -- --exact
```

## TunerStudio round trip

The automated `tunerstudio_metadata_and_quoting_are_serialized` and
`scalar_array_bits_text_round_trip` tests pin UTF-8 output, quoted bit/text
values, row-shaped arrays, and OpenTune reloading. Both real projects were
created and saved by TunerStudio, and OpenTune loaded them without failed
assignments.

The cross-application acceptance passed with OpenTune v0.2.0 and the installed
TunerStudio MS Ultra 3.3.01 on a disposable copy of `MS3-Example_Project`:

| Value | Source project | OpenTune edit | TunerStudio re-save | OpenTune reopen |
| --- | --- | --- | --- | --- |
| `enginesize` | `2000` | `1998` | `1998.0` | `1998` |
| `sequential` | `Off` | `Semi-sequential` | `Semi-sequential` | `Semi-sequential` |
| `veTable1` at 400 kPa / 500 rpm | `90.0` | `91.0` | `91.0` | `91.0` |

The OpenTune output SHA-256 was
`0f9f45f0cc7b46db71e89fc63e479200c843e9f9e98b1dac57f963d9df7a6e6d`;
the TunerStudio re-save SHA-256 was
`10db09b4507bfeaad5b50664693001c95f1217d893806406567fe911a255d927`.
TunerStudio reported zero invalid-table errors and preserved all three values.
Its audit log contained 24 warning-only unit mismatches where the MS3 INI uses
dynamic unit expressions; none of the three acceptance values was lost.

TunerStudio's Swing controls were not exposed through the macOS accessibility
tree, so the installed application's real project loader and MSQ writer were
invoked by a temporary local reflection harness while OpenTune's two legs were
verified in its built GUI. No third-party project or generated tune was added to
the repository.

## Explicit non-claims

- No Honda OBD1 or BMW MS4x definitions/captures were available, so M6 does not
  claim either family.
- These checks use files and the built-in simulator, not a physical ECU. They do
  not establish real-hardware burn safety.
- Unsupported TunerStudio presentation widgets listed in diagnostics are not
  silently represented as working OpenTune controls.

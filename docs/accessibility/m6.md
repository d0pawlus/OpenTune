# M6 accessibility evidence

Checked on 2026-07-18 on macOS. This is a baseline for the application shell,
first-run guide, updater notice, and shared interaction tokens; it is not a
claim of full WCAG conformance for every firmware-generated control.

## Automated checks

| Area | Evidence | Result |
| --- | --- | --- |
| First-run shell | `axe-core` scan with onboarding dialog open; computed color contrast excluded because jsdom has no canvas | No reported violations |
| Returning-user shell | `axe-core` scan after onboarding completion; same contrast exclusion | No reported violations |
| Dialog focus and isolation | Initial focus, Tab/Shift+Tab containment, Escape close, focus restoration, and inert background tests | Pass |
| Preferences | English/Polish, default/high-contrast theme, completion, invalid values, and unavailable storage | Pass |
| Touch/keyboard targets | Shared controls use a 44 px minimum target and visible `:focus-visible` outline token | Implemented |
| Reduced motion | Shared transitions are disabled for `prefers-reduced-motion: reduce` | Implemented |
| Translation coverage | Every English and Polish key is non-empty; M6 shell families resolve in both locales | Pass |

The focused frontend suite contains the onboarding, preference, updater, shell,
i18n, and axe regression tests. The complete frontend suite and the Rust
workspace are also part of the M6 release gate.

## Manual checks

| Check | Status | Notes |
| --- | --- | --- |
| Keyboard-only first run and reopen | Pending | macOS blocked the Computer Use driver while Accessibility/Screen Recording consent remained pending |
| High-contrast visual smoke | Pending | Theme state and tokens are test-pinned; visual inspection still requires the UI driver |
| VoiceOver reading order and control names | Pending | Not represented as passed without an assistive-technology run |

These manual items are the final UI acceptance gate for M6. Findings should be
recorded here with the tested build, macOS version, and any follow-up issue.

## Known boundaries

- Firmware-defined dialogs can contain combinations not present in the shell
  fixtures; they need targeted checks as compatibility expands.
- Canvas gauges expose surrounding controls but are primarily visual; richer
  non-visual realtime summaries remain future work.
- Unsupported TunerStudio `UserDefined` widgets are reported as compatibility
  degradations rather than rendered as inaccessible approximations.

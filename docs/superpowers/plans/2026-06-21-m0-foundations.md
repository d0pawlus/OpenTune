# M0 — Foundations Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A buildable, runnable, empty-but-real Tauri v2 desktop app — Rust
workspace + React/TS frontend — with typed IPC (one generated command + one
generated event), i18n + theming scaffolding, linting, and green cross-platform CI.

**Architecture:** A Tauri v2 shell with a Rust backend organized as a Cargo
workspace of empty domain crates (locking in the module boundaries from
ARCHITECTURE §5) and a React + TypeScript + Vite frontend. The IPC contract is
**generated from Rust** via `tauri-specta` so TypeScript types can never drift from
the backend — proving the single most important architectural guardrail before any
feature code lands.

**Tech Stack:** Tauri v2, Rust (stable), React 18 + TypeScript + Vite, Zustand,
`tauri-specta` + `specta` (type generation), Vitest + Testing Library, GitHub
Actions.

## Global Constraints

These apply to **every task** below. Values copied from
[ARCHITECTURE.md](../../ARCHITECTURE.md),
[the ADRs](../../adr/), and the user's web coding-style rules.

- **App shell:** Tauri **v2** (not v1). [ADR-0001]
- **Backend:** Rust, stable toolchain, organized as a Cargo workspace under
  `src-tauri/` (`crates/` + the app package). [ARCHITECTURE §5, §14]
- **Frontend:** React + TypeScript + **Vite**; **Zustand** for state. [ADR-0003]
- **IPC types are generated from Rust — never hand-duplicated.** [ADR-0003, ADR-0007]
- **License:** every source file carries `SPDX-License-Identifier: GPL-3.0-or-later`. [ADR-0005]
- **Privacy:** offline-first; **no network telemetry** of any kind. [ARCHITECTURE §12]
- **i18n:** UI strings externalized from day one; **Polish and English** first. [ARCHITECTURE §12]
- **Targets:** macOS arm64 (first-class) + x64, Windows x64, Linux x64. [README]
- **File size:** prefer small focused files (<400 lines typical); organize by
  feature, not by type. [web coding-style]
- **Commits:** conventional-commit format (`feat:`, `chore:`, `ci:`, `docs:`),
  frequent, one per task step where indicated.

### Task 4: Typed demo event + Zustand store

**Files:**
- Create: `src-tauri/src/events.rs`, `src/stores/connection.ts`
- Modify: `src-tauri/src/lib.rs` (register event + emit loop), `src/App.tsx`
- Test: `src/stores/connection.test.ts`

**Interfaces:**
- Consumes: the builder from Task 3.
- Produces: a typed event `Heartbeat { seq: u32 }` emitted ~1 Hz, and a Zustand
  store `useConnectionStore` with `{ lastSeq: number | null, setSeq(n: number): void }`.

- [ ] **Step 1: Define and register the event in Rust**

Add `src-tauri/src/events.rs`:

```rust
// SPDX-License-Identifier: GPL-3.0-or-later
use serde::Serialize;
use specta::Type;
use tauri_specta::Event;

#[derive(Serialize, Type, Clone, Event)]
pub struct Heartbeat {
    pub seq: u32,
}
```

In `src-tauri/src/lib.rs`: add `mod events;`, register events on the builder with
`.events(tauri_specta::collect_events![events::Heartbeat])`, mount with
`builder.mount_events(app);` inside `.setup(...)`, and spawn a 1 Hz emit loop:

```rust
use tauri_specta::Event as _;
// inside .setup(|app| { ... }):
let handle = app.handle().clone();
std::thread::spawn(move || {
    let mut seq = 0u32;
    loop {
        std::thread::sleep(std::time::Duration::from_secs(1));
        seq += 1;
        let _ = events::Heartbeat { seq }.emit(&handle);
    }
});
```
